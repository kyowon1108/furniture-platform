# 개발 가이드

## 사전 준비

- Python 3.10 or 3.11 recommended
- Node.js 20+
- npm 10+
- 선택사항: Docker / Docker Compose (PostgreSQL 로컬 실행 시)

## 빠른 시작

### 1. Backend

```bash
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
alembic upgrade head
uvicorn app.main:socket_app --reload --host 0.0.0.0 --port 8008
```

### 2. Frontend

```bash
cd frontend
npm install
npm run dev
```

### 3. 접속 확인

- Frontend: `http://localhost:3008`
- Backend API: `http://localhost:8008/api/v1`
- Swagger UI: `http://localhost:8008/docs`

## 환경 변수

### Backend

필수 또는 자주 쓰는 값:

- `DATABASE_URL`
- `SECRET_KEY`
- `ALLOWED_ORIGINS`
- `ADMIN_EMAILS`
- `AWS_ACCESS_KEY_ID`
- `AWS_SECRET_ACCESS_KEY`
- `S3_BUCKET_NAME`

`ADMIN_EMAILS`는 카탈로그 관리 API에 접근할 이메일 allowlist입니다.

### Frontend

- `NEXT_PUBLIC_API_URL=http://localhost:8008/api/v1`
- `NEXT_PUBLIC_SOCKET_URL=http://localhost:8008`

프론트는 인증 토큰을 `sessionStorage`에 저장하고, 기존 `localStorage` 토큰이 있으면 자동으로 마이그레이션합니다.

## 로컬 DB 옵션

### SQLite 빠른 시작

- `backend/dev.db` 파일 기반
- 가장 빠른 개발용 시작 경로

### PostgreSQL 로컬 실행

```bash
docker compose up -d postgres
```

예시 연결 문자열:

```env
DATABASE_URL=postgresql://furniture:furniture123@127.0.0.1:5433/furniture_db
```

## 검증 순서

### Backend

```bash
cd backend
pytest
```

선택 의존성이 없는 core 테스트만 먼저 돌리려면:

```bash
cd backend
pytest tests/test_auth.py tests/test_collision.py tests/test_catalog.py tests/test_projects.py tests/test_files.py
```

`backend/test_mesh_generation.py`는 `open3d`가 있을 때만 의미 있는 optional 테스트입니다.


## 개발 시 주의점

- 프로젝트 응답에는 내부 파일 경로가 포함되지 않습니다. 파일 접근은 `download_url` 또는 보호된 다운로드 엔드포인트를 사용해야 합니다.
- Socket.IO 연결은 JWT가 없으면 거부됩니다.
- 카탈로그 쓰기/동기화 API는 `ADMIN_EMAILS` 설정이 없으면 사용할 수 없습니다.
- `docs/update_code/`는 변경 보고서 아카이브이며, 운영 기준 문서는 `README.md`, `docs/README.md`, `docs/05_API_문서.md`, 이 문서, `deployment/README.md`입니다.
- 제출/면접용 설명은 `docs/09_기술_의사결정.md`, `docs/10_개인_기여_및_면접_QA.md`를 함께 보는 것이 좋습니다.

## 배포 메모

- 배포 템플릿은 HTTPS 전제를 기준으로 작성됩니다.
- Nginx는 `/uploads/`를 직접 공개하지 않습니다.
